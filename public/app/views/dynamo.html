Author's note: using an integer primary key in dynamo is not wise. DynamoDB requires you to pass in a unique id for each new entry, so if you don't know what's been used already, it might be tough to figure out what this should be. It would be smarter to use a hash value for the primary key instead. I didn't realize that when I set this up, and since it's small, I can deal with it manually.
